# 重要告示 <!-- omit in toc -->

> [!重要的]\
> 由于 GitHub API 仅 [允许每个用户帐户每小时发送 5000 个请求](https://docs.github.com/en/graphql/overview/resource-limitations), 因此托管的公共 Vercel 实例`https://github-readme-stats.vercel.app/api`可能会触发速率限制器 (见 [#1471](https://github.com/anuraghazra/github-readme-stats/issues/1471)).我们使用缓存来防止这种情况发生 (见 https://github.com/anuraghazra/github-readme-stats#common-options).你可以通过部署 [自己的Vercel 实例](#disable-rate-limit-protections)来关闭速率限制.

<img alt="Uptime Badge" src="https://img.shields.io/endpoint?url=https%3A%2F%2Fgithub-readme-stats-git-monitoring-github-readme-stats-team.vercel.app%2Fapi%2Fstatus%2Fup%3Ftype%3Dshields">

> [!重要的]\
> 我们团队规模不大，依靠点赞👍来确定优先级。我们使用 Top Issues 仪表盘来追踪社区需求 (见 [#1935](https://github.com/anuraghazra/github-readme-stats/issues/1935)).请随时为您感兴趣的 Issue 和 Pull Request 点赞。我们会优先处理点赞最多的。

# GitHub 统计卡

复制并粘贴到你的 markdown 中，就完成了。简单！

将值更改 `?username=` 为您的 GitHub 用户名。

```md
[![Anurag's GitHub stats](https://github-readme-stats.vercel.app/api?username=anuraghazra)](https://github.com/anuraghazra/github-readme-stats)
```

> [!警告]\
> 默认情况下，统计信息卡片仅显示来自公共代码库的星标、提交和拉取请求等统计信息。如需在统计信息卡片上显示私有统计信息，您需要使用自己的 GitHub API 令牌 [部署您自己的实例](#deploy-on-your-own).

> [!NOTE]\
> 可用的排名包括 S（前 1%）、A+（12.5%）、A（25%）、A-（37.5%）、B+（50%）、B（62.5%）、B-（75%）、C+（87.5%）和 C（所有人）。此排名方案基于 [日本的学术评分系统](https://wikipedia.org/wiki/Academic_grading_in_Japan).[全局百分位数是根据指数分布](https://wikipedia.org/wiki/exponential_distribution) 和 [对数正态分布](https://wikipedia.org/wiki/Log-normal_distribution) 的累积分布函数，对每个统计数据（提交数、拉取请求数、评论数、问题数、星标数和关注者数）的百分位数加权和计算得出的. 可以在 [src/calculateRank.js](https://github.com/anuraghazra/github-readme-stats/blob/master/src/calculateRank.js)中查看其实现. 排名周围的圆圈表示 100 减去全局百分位数。

### 隐藏个人数据

您可以传递查询参数 `&hide=` 来隐藏任何以逗号分隔的值的特定统计数据。

> 选项: `&hide=stars,commits,prs,issues,contribs`

```md
![Anurag's GitHub stats](https://github-readme-stats.vercel.app/api?username=anuraghazra&hide=contribs,prs)
```

### 显示更多个人统计数据

您可以传递查询参数`&show=`以使用逗号分隔的值显示任何特定的附加统计数据。

> 选项: `&show=reviews,discussions_started,discussions_answered,prs_merged,prs_merged_percentage`

```md
![Anurag's GitHub stats](https://github-readme-stats.vercel.app/api?username=anuraghazra&show=reviews,discussions_started,discussions_answered,prs_merged,prs_merged_percentage)
```

### 显示图标

要启用图标，您可以`&show_icons=true`传入查询参数，如下所示：

```md
![Anurag's GitHub stats](https://github-readme-stats.vercel.app/api?username=anuraghazra&show_icons=true)
```

### 主题

使用内置主题，您可以自定义卡片的外观，而无需进行任何 [手动自定义](#customization).

使用 `&theme=THEME_NAME` 如下参数：

```md
![Anurag's GitHub stats](https://github-readme-stats.vercel.app/api?username=anuraghazra&show_icons=true&theme=radical)
```

# 常用语言统计

常用语言统计卡片显示 GitHub 用户最常用的语言。

> [!WARNING]\
> 默认情况下，语言卡片仅显示来自公共仓库的语言结果。如需包含私有仓库中使用的语言，您需要使用[自己的 GitHub API 令牌部署您自己的实例。](#deploy-on-your-own)。

> [!NOTE]\
> 热门语言并不代表用户的技能水平或类似的东西；它是 GitHub 的一个指标，用于确定哪些语言在 GitHub 上拥有最多的代码。它是 github-readme-stats 的一个新功能。

> [!WARNING]\
> 此卡片仅显示您自己未分叉的仓库中的语言使用情况，与提交的作者无关。它不包含您对其他用户/组织仓库的贡献。目前无法通过 GitHub API 获取此数据。如果您希望改进此行为，可以支持[@rickstaa](https://github.com/rickstaa)在 GitHub 社区中提出的 [请求](https://github.com/orgs/community/discussions/18230)

> [!WARNING]\
> 目前此卡片仅显示前 100 个仓库的数据。这是因为 GitHub API 的限制导致公共实例宕机 (见 [#1471](https://github.com/anuraghazra/github-readme-stats/issues/1471)). 未来将通过发布 GitHub Action 或为用户自己的实例提供环境变量来改进此行为。

### 用法

### 语言统计算法

我们使用以下算法来计算语言卡上的语言百分比：

```js
ranking_index = (byte_count ^ size_weight) * (repo_count ^ count_weight)
```

默认情况下，仅使用字节数来确定语言卡上显示的语言百分比(即`size_weight=1`和`count_weight=0`)）。但是，您可以使用`&size_weight=`和`&count_weight=`选项来加权语言使用情况的计算。值必须是正实数[有关该算法的更多详细信息，请参见此处](https://github.com/anuraghazra/github-readme-stats/issues/1600#issuecomment-1046056305).

*   `&size_weight=1&count_weight=0` - *(默认)* 按字节数排序.
*   `&size_weight=0.5&count_weight=0.5` - *(推荐)* 使用字节数和 repo 数进行排名
*   `&size_weight=0&count_weight=1` - 按回购数量排序
```md
![Top Langs](https://github-readme-stats.vercel.app/api/top-langs/?username=anuraghazra&size_weight=0.5&count_weight=0.5)
```

# 自行部署

## 在 Vercel 上

### :film\_projector: [查看@codeSTACKr 的分步视频教程](https://youtu.be/n6d4KHSKqGk?t=107)

由于 GitHub API 每小时仅允许 5000 个请求，因此`https://github-readme-stats.vercel.app/api`可能会达到速率限制。如果您将其托管在自己的 Vercel 服务器上，则无需担心任何事情。点击部署按钮即可开始！

> [!NOTE]\
> 从 [#58](https://github.com/anuraghazra/github-readme-stats/pull/58)开始，我们应该能够处理超过 5k 个请求，并且停机问题更少 :grin:.

> [!NOTE]\
> 如果您使用的是Vercel [Pro (付费版)](https://vercel.com/pricing) 方案。当您的 Vercel 实例在卡片请求期间频繁超时时，可以增加[maxDuration](https://vercel.com/docs/concepts/projects/project-configuration#value-definition)中的[vercel.json](https://github.com/anuraghazra/github-readme-stats/blob/master/vercel.json)值，建议您将此值保持在低于`30`的范围内，以防止内存占用过高。

[![Deploy to Vercel](https://vercel.com/button)](https://vercel.com/import/project?template=https://github.com/anuraghazra/github-readme-stats)

<details>
 <summary><b>:hammer_and_wrench: 设置您自己的 Vercel 实例的分步指南</b></summary>

7.  要导入项目，请单击`Add New...`按钮并选择`Project`选项。
    ![](https://files.catbox.moe/3n76fh.png)
8.  点击`Continue with GitHub`按钮，搜索所需的 Git 仓库并点击按钮导入。或者，您也可以使用页面底部的链接 `Import`导入第三方 Git 仓库。`Import Third-Party Git Repository ->`
    ![](https://files.catbox.moe/mg5p04.png)
